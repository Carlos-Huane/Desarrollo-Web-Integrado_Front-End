 CÓMO LEVANTAR EL ENTORNO (mismo para ambos)
================================================================================

REQUISITOS
----------
1. Node.js 20+
2. JDK 21 (https://adoptium.net)
3. Maven 3.9+
4. MySQL 8 corriendo en localhost:3306
   - Usuario: root
   - Password: admin (o lo que tengas configurado)
5. Angular CLI 18 (instalar con: npm i -g @angular/cli@18)

REPOS
-----
- Backend:   https://github.com/Carlos-Huane/Desarrollo-Web-Integrado_Back-End-
- Frontend:  https://github.com/Carlos-Huane/Desarrollo-Web-Integrado_Front-End

PASO 1 — CLONAR
---------------
git clone https://github.com/Carlos-Huane/Desarrollo-Web-Integrado_Back-End-.git back
git clone https://github.com/Carlos-Huane/Desarrollo-Web-Integrado_Front-End.git  front
cd back  && git checkout develop && cd ..
cd front && git checkout develop && cd ..

PASO 2 — LEVANTAR BACKEND (terminal 1)
--------------------------------------
cd back
# si tu password de MySQL no es "admin", editar src/main/resources/application.properties
mvn spring-boot:run

Esperar a ver "Started IntranetApplication in X seconds".
El backend escucha en http://localhost:8080.

PASO 3 — LEVANTAR FRONTEND (terminal 2)
----------------------------------------
cd front
npm install     (solo la primera vez)
ng serve

Esperar a ver "Local: http://localhost:4200/".

PASO 4 — ABRIR EL NAVEGADOR
---------------------------
http://localhost:4200

================================================================================
 PREGUNTAS FRECUENTES
================================================================================

¿"npm install" se queda colgado?
  → Asegúrate de tener Node 20+. Borra node_modules y package-lock.json y
    vuelve a intentar.

¿El back no levanta porque no encuentra MySQL?
  → Verifica que MySQL esté corriendo en localhost:3306. La BD `intranet_db`
    se crea sola, no necesitas crearla tú. El password root está en
    `application.properties` (por defecto "admin").

¿Cómo abro la consola del navegador?
  → F12 o Ctrl+Shift+I. Pestaña "Console" para errores JS, pestaña "Network"
    para ver llamadas HTTP.

¿Cómo simulo móvil para CP-57/58?
  → F12 → click en el ícono de dispositivo (Ctrl+Shift+M en Chrome) →
    elegir iPhone, iPad o ajustar tamaño manual.

¿Algo me bloquea y no puedo seguir?
  → Marca el caso como "Bloqueado", reporta el problema y sigue con el
    siguiente. No te quedes esperando.
